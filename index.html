Integration Diagram  ·  @type id

Models an IT architecture landscape — systems, databases, and queues, how they connect, and their lifecycle state. Each element is coloured by platform (AWS, Azure, GCP, Oracle, on-prem). State is shown through fill transparency and border style.

@type     id
@name     My Platform
@version  1.0
@author   Team
@theme    dark | light | tokyo
@size     a4 | a3 | a2 | a1 | a0
@legend   on | off               # show/hide legend box (default: on)

system    <id>  [aws|azure|gcp|oracle|on-prem]  [new|changing|decommissioned]
database  <id>  [aws|azure|gcp|oracle|on-prem]  [new|changing|decommissioned]
queue     <id>  [aws|azure|gcp|oracle|on-prem]  [new|changing|decommissioned]

# Label placement override (default: system=inside, database/queue=below)
system    <id>  [platform]  [label:below]
database  <id>  [platform]  [label:inside]

connect  <from>  ->   <to>  : <protocol>    # unidirectional
connect  <from>  <->  <to>  : <protocol>    # bidirectional

group <id> <label> [label:upper-left|upper-right|lower-left|lower-right]
  system/database/queue declarations...
end

Information Flow Diagram  ·  @type infoflow

Models a data landscape — where data lives, what role each store plays, and how data moves between stores. Each store is coloured by its role (master, replica, derived, etc.). Relationships describe the nature of the data flow.

@type     infoflow
@name     My Data Landscape
@version  1.0
@author   Team
@theme    dark | light | tokyo
@size     a4 | a3 | a2 | a1 | a0

store <id>  [master|replica|derived|aggregate|golden|reference|consumer]
            [new|changing|decommissioned]
            [label:"Human Readable Name"]

# Relationships: replicate publish ingest derive aggregate enrich merge serve
link  <from>  ->  <to>  : <relationship>  [transport]

group <id> <label> [label:upper-left|upper-right|lower-left|lower-right]
  store declarations...
end
