System Dynamics  ·  @type sd

Models dynamic systems using Forrester stock-and-flow notation — stocks accumulate, flows transfer between stocks, auxiliaries compute intermediate values, and connectors show causal influence. This is the default diagram type.

@type     sd                       # optional — sd is the default
@name     My Model
@version  1.0
@author   Team
@theme    dark | light | tokyo
@size     a4 | a3 | a2 | a1 | a0
@legend   on | off               # show/hide legend box (default: on)

stock  <name>
cloud  <name>  [source|sink]

# Flows transfer between stocks/clouds
flow   <from>  ->  <to>  : <label>  (+|-)  [weak|medium|strong]

# Auxiliaries compute intermediate values
aux    <name>
aux    <name>  <-  <source> (+|-)              # with inflow
aux    <name>  <-  <src1> (+), <src2> (-)      # multiple inflows

# Connectors show causal influence
connector  <to>  <-  <from> (+|-)
connector  <to>  <-  <src1> (+), <src2> (-)    # multiple sources

# Grouping (stocks and auxiliaries only — not clouds)
group <id> <label> [corner:upper-left|upper-right|lower-left|lower-right]
  stock/aux declarations...
end

Integration Diagram  ·  @type id

Models an IT architecture landscape — systems, databases, and queues, how they connect, and their lifecycle state. Define your own location-types (e.g. aws, legacy, cloud) mapped to palette colours. State is shown through fill transparency and border style.

@type     id
@name     My Platform
@version  1.0
@author   Team
@theme    dark | light | tokyo
@size     a4 | a3 | a2 | a1 | a0
@legend   on | off               # show/hide legend box (default: on)

# Define location-types with palette colours
@location-types
  aws orange
  azure blue
  gcp green
  on-prem grey
  legacy green
  cloud blue

# Palette colours: green blue red orange purple grey yellow cyan pink teal

system    <id>  [<location-type>]  [new|changing|decommissioned]
database  <id>  [<location-type>]  [new|changing|decommissioned]
queue     <id>  [<location-type>]  [new|changing|decommissioned]

# Label placement override (default: system=inside, database/queue=below)
system    <id>  [<location-type>]  [placement:below]
database  <id>  [<location-type>]  [placement:inside]

connect  <from>  ->   <to>  : <protocol>    # unidirectional
connect  <from>  <->  <to>  : <protocol>    # bidirectional

group <id> <label> [corner:upper-left|upper-right|lower-left|lower-right]
  system/database/queue declarations...
end

Information Flow Diagram  ·  @type infoflow

Models a data landscape — where data lives, which processes act on it, and how information moves across stores and processes. Stores are coloured by location-type, processes by system, and links by flow-type.

@type     infoflow
@name     My Data Landscape
@version  1.0
@author   Team
@theme    dark | light | tokyo
@size     a4 | a3 | a2 | a1 | a0

# Define store location-types with palette colours
@location-types
  master blue
  replica cyan
  derived green
  aggregate purple
  golden orange
  reference grey
  consumer grey

# Palette colours: green blue red orange purple grey yellow cyan pink teal

@systems
  SystemA blue
  SystemB teal

@flow-types
  sync solid
  async dashed
  batch thick

store    <id>  [<location-type>]
               [unchanged|new|changing|decommissioned]
               [label:"Human Readable Name"]

process  <id>  [<system>]
               [unchanged|new|changing|decommissioned]
               [label:"Human Readable Name"]

# State visuals: unchanged=current solid, changing dashed,
#                new dotted, decommissioned = X marker

# Relationships: replicate publish subscribe ingest derive
#                aggregate enrich merge serve query
link  <from>  ->  <to>  : <relationship>  [flow:<type>]

group <id> "<label>" [system:<system>]
      [corner:upper-left|upper-right|lower-left|lower-right]
  store declarations...
  process declarations...
  link declarations...
end

Threat Model  ·  @type tm

Models threats against a system using the STRIDE framework. Refs are ghost representations of elements declared in other diagrams. Threats are annotated with STRIDE category badges; mitigations are listed in a panel.

@type     tm
@name     My Threat Model
@version  1.0
@author   Team
@theme    dark | light | tokyo
@size     a4 | a3 | a2 | a1 | a0
@show-ids on | off               # show element id badges

# Reference elements from other diagram files
@ref  integration_example.id
@ref  customer_information.iff

# Trust boundaries
boundary <id> [label:"Boundary Label"]
  ref <element-id>   # id from a referenced diagram
  ref <element-id>
end

# Refs outside a boundary
ref <element-id>

# Data flows between refs
flow <id> <from> -> <to> [label:"description"]

# STRIDE threats — S T R I D E
threat <id> [stride:S|T|R|I|D|E] <targetId> : "description"

# Mitigations
mitigate <threatId> : "description"

Common directives

@include   common.iff   # load shared defaults/dictionaries from the Config folder
@show-ids  on | off    # overlay [id] badge on each element (all diagram types)
