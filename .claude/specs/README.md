# Diagram Specifications

This directory contains durable, agent-readable specifications for each diagram type.

Use these files for formal syntax, semantic validation rules, rendering expectations, examples, invalid cases, and required tests:

- `stock-flow.md`
- `integration-diagram.md`
- `information-flow-diagram.md`
- `threat-model.md`

Keep `.claude/requirements.md` product-level. Keep implementation rules in `.claude/rules/`. Keep completed plans and migration notes in `.claude/history/`.
